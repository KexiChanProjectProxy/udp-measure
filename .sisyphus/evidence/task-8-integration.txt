## Bug Fixed: UDP Receiver Socket Leak
Root cause: The UDP receiver socket was not being closed after use, causing "address already in use" errors on subsequent tests using the same UDP port.

Fix applied:
1. Added `receiver.Close()` in `runUplinkReceiver` after `Receive` returns
2. Added UDP receiver cleanup in `cleanup()` and `cleanupNoStateChange()` functions
